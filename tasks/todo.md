# Add sample guide CTA to landing page

**Notion spec:** https://www.notion.so/5b1d542237994e8d982cfbf0fac34ba2

Done when:
- A secondary CTA appears below the hero input, above the problem section, linking to the sample guide
- `/example` renders the Apple guide at full visual quality - indistinguishable from a real guide, just read-only
- No auth required, no paywalls, no account UI anywhere on `/example`
- Conversion CTA ("Build yours") is present in the header and sidebar footer
- Disclaimer is on the cover: "Generated by Tone of Voice App from apple.com"
- Primary "Get Started" CTA on the landing page is unchanged and still dominant

---

## Steps

### 1. Generate and capture Apple guide content
- Run apple.com through the tool, generate the full guide
- Copy the guide section data (titles, icons, markdown content) into `/lib/example-guide.ts` as a typed static constant
- Include all sections fully expanded (no locked sections in the sample)
- Set `brandName: "Apple"`, `websiteUrl: "apple.com"`, `date: <generation date>`

---

### 2. Create `/example` route

Add `app/example/page.tsx`. It should:
- Be a fully public Server Component (no auth check, no redirect)
- Import and render the guide using the existing `GuideLayout` + `GuideView` structure
- Pass the static Apple guide data
- Pass `viewMode="preview"` permanently (no edit mode)
- Pass a custom `isUnlocked` that always returns `true` (no ContentGate anywhere)
- Suppress all banners: no expand-guide banner, no post-export prompt, no first-visit tip

---

### 3. Sidebar: strip account/upgrade UI, replace with conversion CTA

The example sidebar should look like the real sidebar but with all account and upgrade elements removed:

**Keep:**
- Brand name ("Apple") + "Tone of Voice Guidelines" subtitle
- Full section navigation list with icons, active state, scroll behavior

**Remove:**
- Tier badge (PRO/AGENCY pill)
- Lock icons on section items
- "Add section" button
- Progress bar ("X of Y sections included")
- "Unlock Full Guide" button
- User account footer (avatar, name, email, dropdown)

**Add (sidebar footer instead):**
- Simple CTA block: "Build your own guide" headline + "Get Started free →" button linking to `/`
- Keep it light — don't make it feel like an ad. Think: a small card, subtle border, same sidebar padding

Options: either pass a `footerContent` prop variant to `GuideSidebar`, or create a thin `ExampleGuideSidebar` wrapper that composes the existing sidebar structure minus the account bits. Prefer the wrapper approach if `GuideSidebar` doesn't already support prop-level customisation cleanly.

---

### 4. Header: strip edit/download tools, add conversion CTA

**Keep:**
- Sidebar trigger (hamburger toggle)
- Separator

**Remove:**
- "Dashboard" back link (irrelevant — not coming from a dashboard)
- Edit/Preview toggle button (`showEditTools={false}`)
- Download button

**Add (right side of header):**
- Single primary button: "Build yours free →" → links to `/`
- Keep it compact (same height as existing header buttons)

---

### 5. Cover page: Apple branding + disclaimer

The cover page renders from `GuideCover`. For the example:

**Keep unchanged:**
- Decorative gradient + grid background
- Eyebrow text: "Brand Voice & Content Guidelines"
- Brand name: "Apple" (giant serif headline)
- Black rule bar
- Date (the generation date, hardcoded in the static fixture)
- Website URL: "apple.com"

**Change:**
- Remove "Preview Mode" badge — this isn't a preview, it's a real guide
- Set branding footer to show "Tone of Voice App" (like pro tier — a small, tasteful link)

**Add:**
- Disclaimer line below or near the website URL: "Generated by Tone of Voice App from apple.com · Not an official Apple document"
- Style it small, gray, italic — factual, not apologetic

---

### 6. Guide content: all sections unlocked, no editing affordances

- No `ContentGate` anywhere — all sections render as plain `MarkdownRenderer`
- No locked badges, no blur overlays, no upgrade prompts within content
- No "click to edit" cursor or edit-mode UI
- Sections render exactly as they would for a pro user viewing their own guide in preview mode

---

### 7. Conversion CTA at the bottom of the guide

After the last section (or just before the "Questions/Contact" section), add a full-width CTA block inside the guide document:

- Headline: "Like what you see? Build your guide in 5 minutes."
- Subtext: "Answer a few questions about your brand. We'll generate a complete tone of voice guide like this one."
- CTA button: "Get Started free →" → links to `/`
- Style: matches the guide's document aesthetic (white/off-white background, same padding rhythm as sections, clean typography — not a marketing banner)

---

### 8. Add secondary CTA to hero section on landing page

Edit `components/landing/hero-section.tsx`:
- Below the form/submit button, add a small text link:
  "Or see a sample guide first →" → links to `/example`
- Style: `text-sm`, muted color (gray-500 or similar), no button chrome
- Placement: directly below the form, above any existing "Add details manually here" link
- Must not compete visually with the primary CTA

---

### 9. Verify

- `/example` loads with no auth, no redirect
- Full guide renders: cover, all sections visible, no locked/blurred content
- No account UI, no upgrade prompts, no edit mode anywhere
- Disclaimer visible on cover
- "Build yours free →" visible in header and sidebar footer
- Bottom conversion block visible after last section
- Landing page: secondary CTA visible below hero form
- Mobile (375px): sidebar collapses, header CTA still accessible, secondary landing CTA readable without crowding the hero

---

## Technical decisions

- **Static fixture** (`/lib/example-guide.ts`) over live Supabase query — no auth complexity, instant load, no risk of data changing
- **`/example` as dedicated route** — not a query param on `/guide`. Cleaner URL to share and link from ads
- **Wrapper approach for sidebar/header** — rather than adding conditional prop branches to `GuideSidebar` for one-off use, create `ExampleGuideSidebar` / `ExampleGuideHeader` thin wrappers. Keeps the real guide components clean
- **`isUnlocked` always returns true** — reuses existing section rendering logic without touching ContentGate code
- **Ungated** — no email capture to start; measure click-through first (per spec)
